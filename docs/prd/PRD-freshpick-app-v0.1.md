# 프레시픽(FreshPick) MVP PRD v0.5

## 🎯 핵심 정보

**목적**: 동네 슈퍼·정육점·반찬가게·과일가게의 신선식품·생필품을 온라인으로 주문·결제해 즉시 배송받는 동네 마켓 플랫폼
**사용자**: 근거리 신선식품을 빠르고 편리하게 받아보고자 하는 20-40대 동네 거주자

---

## 🎨 UX 디자인 원칙

### 밀도 레이어 (Density Layer)

프레시픽 사용자는 **오프라인 구매 경험 40% · 온라인 구매 경험 60%** 비율로 구성되며 온라인 구매자가 점증하는 특성을 가진다. 두 그룹 모두를 만족시키는 "고밀도 + 직관적" UI를 기본 원칙으로 삼는다.

- **오프라인 친숙 사용자**: 마트 진열대처럼 한 화면에서 많은 상품을 한눈에 훑을 수 있는 고밀도 2열 컨테이너 그리드
- **온라인 구매자**: 핵심 정보를 압축한 컨테이너 + 빠른 인터랙션 + 개인화 피드
- **설계 규칙**: `shadow 0–1dp`, `border 1px neutral-200 이하 또는 투명`, `gap 8–16px spacing으로 항목 분리` — Material Card 대신 **Container Chunk** 패턴 적용

### 인터랙션 레이어 (Interaction Layer)

| 트리거 | 반응 |
|--------|------|
| press / hover | `scale(0.97)` + 배경 톤 변화 + 아이콘 색 변화 |
| 장바구니 담기 버튼 | 상품 이미지가 하단 탭 카트 아이콘으로 날아가는 Fly 애니메이션 + Toast "장바구니에 담았어요" |
| 항목 좌측 스와이프 | 삭제 액션 노출 (장바구니·찜·메모 공통) |
| 항목 우측 스와이프 | 찜 토글 (찜 페이지·장바구니 페이지) |
| 세로 스크롤 | 헤더·탭·플로팅 버튼 `sticky` 유지, 블러/그림자 전환 |
| 가로 스와이프 | 상품 이미지 캐러셀·프로모션 배너·카테고리·탭 전환 |

### 개인화 레이어 (Personalization Layer)

- **F014 AI 추천 장보기**: 과거 주문·찜·시즌 컨텍스트 기반 AI 주간 장보기 리스트 자동 생성
- **F015 지난 구매 자동 리스트**: 재구매 주기(요일·간격) 분석 → "매주 화요일 사는 우유" 자동 큐레이션
- **개인화 피드**: 메인 홈 상단 "OO님을 위한 추천" 섹션 — 최근 찜·조회 상품 기반

## 🚶 사용자 여정

```
1. 로그인 페이지
   ↓ [이메일 로그인] 성공 → 프레시픽 메인 페이지
   ↓ [SNS 간편 로그인] 카카오·네이버·구글·페이스북·애플 → 최초 가입 시 약관동의 페이지, 기존 회원 → 프레시픽 메인 페이지
   ↓ [신규 회원] 회원가입 버튼 클릭

2. 회원가입 페이지
   ↓ 이메일 정보 입력 또는 SNS 계정 선택 후 다음
   → 약관동의 페이지 → 동의 완료
   → 개인인증(PASS) 페이지 → 인증 완료
   ↓ [가입 완료]

3. 프레시픽 메인 페이지
   ↓ [개인화 피드] → 추천 상품 즉시담기 (Fly 애니메이션 + Toast)
   ↓ [탐색 분기]
   [AI 장보기 탭] → 장보기 메모 페이지
      ├── [AI 추천] 탭 → AI 추천 리스트 → 원클릭 전체/선택 장바구니 담기 (F014)
      ├── [자동 리스트] 탭 → 지난 구매 자동 리스트 → 원클릭 재주문 (F015)
      └── [내 메모] 탭 → 수동 메모 작성 후 상품 검색
   [이번주 세일 배너] → 이번주 세일 페이지 (sticky 탭) → 상품 상세 페이지
   [N+1 행사 배너] → N+1 행사 페이지 (sticky 탭) → 상품 상세 페이지
   [랭킹 배너] → 랭킹 페이지 (sticky 탭) → 상품 상세 페이지
   [찜 탭] → 찜 페이지 (좌측 스와이프 삭제 / 우측 스와이프 장바구니)
   ↓

4. 상품 상세 페이지
   ↓ [장바구니 담기 / 찜하기]

5. 장바구니 페이지
   ↓ [주문하기] 버튼 클릭

6. 주문하기 페이지
   ↓ [결제하기] 버튼 클릭

7. 결제하기 페이지
   ↓ [결제 완료]

8. 주문내역 페이지 (배송 상태 확인)
   ↓ [마이프레시 탭에서 접근 가능]

9. 마이프레시 페이지 (주문내역 · 배송지 관리 · 로그아웃)
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 상품 탐색 | 메인 피드, 이번주 세일, N+1 행사, 랭킹으로 상품을 발견하는 기능 | 구매 전환의 시작점, 핵심 상품 노출 채널 | 프레시픽 메인 페이지, 이번주 세일 페이지, N+1 행사 페이지, 랭킹 페이지 |
| **F002** | 상품 상세 조회 | 상품 이미지·가격·영양정보·원산지·알러지·조리팁 확인 | 신선식품 구매 결정의 핵심 정보 제공 | 상품 상세 페이지 |
| **F003** | 장보기 메모 | 사용자 수기 메모 · AI 추천 · 지난 구매 자동 리스트 3-레이어로 쇼핑 목록을 구성하고 장바구니에 담는 핵심 차별 기능 | 장보기 리스트 기반 구매 패턴 지원, 핵심 차별 기능 | 장보기 메모 페이지 |
| **F004** | 장바구니 관리 | 상품 수량 변경·삭제(좌측 스와이프)·소계 계산·쿠폰 적용 | 구매 전 최종 확인 및 조정, 구매 완료의 필수 단계 | 장바구니 페이지, 장보기 메모 페이지 |
| **F005** | 찜(위시리스트) | 원하는 상품을 찜하여 저장·관리, 좌측 스와이프 삭제·우측 스와이프 장바구니 담기 제스처 지원 | 재구매 유도 및 사용자 앱 재방문 동기 부여 | 찜 페이지, 상품 상세 페이지, 프레시픽 메인 페이지 |
| **F006** | 주문/결제 | 배송지 선택·포인트 사용·쿠폰 적용 후 토스페이먼츠 SDK로 결제창 호출(웹뷰/네이티브), `requestBillingAuth`로 자동결제 처리 | 매출 발생의 핵심 기능, 서비스의 최종 목적 | 주문하기 페이지, 결제하기 페이지 |
| **F007** | 주문 내역 및 배송 추적 | 주문 상태 확인(접수→포장→배송중→배송완료) 및 배송 이벤트 조회 | 주문 후 신뢰 확보 및 고객 CS 최소화 | 주문내역 페이지 |

| **F014** | AI 추천 장보기 | 과거 구매 이력·시즌·가구 구성 기반 AI 주간 장보기 리스트 자동 제안. 원클릭 전체/선택 장바구니 담기 지원 | AI 기반 개인화로 구매 편의성 극대화, 오프라인→온라인 전환 촉진 | 장보기 메모 페이지, 프레시픽 메인 페이지 |
| **F015** | 지난 구매 자동 리스트 | 재구매 주기(요일·간격) 분석 → "매주 화요일 사는 우유" 자동 큐레이션 및 원클릭 재주문 | 반복 구매 편의성 향상, 이탈 방지 | 장보기 메모 페이지, 마이프레시 페이지 |
