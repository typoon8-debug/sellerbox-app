# 온라인쇼핑몰 관리자 시스템 MVP PRD

## 🎯 핵심 정보

**목적**: 온라인 쇼핑몰에 입점한 여러 가게(입점사·가맹점)를 테넌트 단위로 통합 관리하는 시스템 관리자 전용 백오피스
**사용자**: 쇼핑몰 플랫폼 운영팀의 시스템 관리자

## 🚶 사용자 여정

```
1. 관리자 로그인 페이지
   ↓ 이메일/비밀번호 입력 후 "로그인" 버튼 클릭

2. [분기] 인증 결과 확인
   [실패] → 에러 메시지 표시 → 로그인 페이지 유지
   [성공] → 테넌트 관리 페이지 (기본 랜딩)
   ↓

3. 좌측 내비게이션에서 관리 업무 선택

   [분기 A] 테넌트 관리 페이지
           → 테넌트 목록 조회 → 등록/수정/삭제

   [분기 B] 사용자 관리 페이지
           → 테넌트 선택 → 사용자 목록 조회 → 등록/수정/삭제

   [분기 C] 코드 관리 페이지
           → 코드 마스터 선택 → 코드값 목록 조회 → 등록/수정/삭제

   [분기 D] 광고 관리 페이지
           → 테넌트 선택 → 광고 목록 조회 → 등록(이미지 업로드)/수정/삭제/활성화 토글
   ↓

4. [완료] → 다른 메뉴로 이동 또는 로그아웃
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 테넌트 CRUD 관리 | 입점사·가맹점을 테넌트로 등록, 수정, 삭제하고 목록을 조회 | 멀티테넌트 시스템의 근간 데이터 | 테넌트 관리 페이지 |
| **F002** | 테넌트별 사용자 CRUD 관리 | 선택한 테넌트에 소속된 사용자를 등록, 수정, 삭제하고 목록을 조회 | 테넌트 운영을 위한 필수 사용자 관리 | 사용자 관리 페이지 |
| **F003** | 공통코드 마스터·상세값 CRUD 관리 | 시스템 전체에서 참조하는 코드 마스터와 코드값을 등록, 수정, 삭제 | 전 시스템 참조 데이터의 통합 관리 | 코드 관리 페이지 |
| **F004** | 테넌트별 광고 CRUD 관리 | 테넌트별로 광고 위치, 광고 이미지, 활성화 여부를 등록, 수정, 삭제 | 쇼핑몰 광고 운영의 핵심 관리 기능 | 광고 관리 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F010** | 시스템 관리자 인증 | Supabase Auth 기반 이메일/비밀번호 로그인·로그아웃 | 모든 관리 기능 접근의 전제 조건 | 관리자 로그인 페이지 |
| **F011** | 광고 이미지 업로드 | Supabase Storage를 사용한 광고 이미지 업로드 및 미리보기 | F004 광고 등록/수정의 필수 하위 기능 | 광고 관리 페이지 |
| **F012** | 내 정보/권한 설정 | 로그인한 관리자가 자신의 이름·전화번호를 수정하고 비밀번호를 변경 | 관리자 개인 계정 관리 | 설정 아이콘 팝오버 → 다이얼로그 |

### 3. MVP 이후 기능 (제외)

- 상세 프로필 관리 (아바타, 자기소개 등)
- OAuth 토큰·디바이스 관리 UI
- OTP 인증·계좌 인증·로그인 기록 조회 화면
- 알림(notification)·사용 동의(consent)·감사 로그(audit_log) UI
- 메시지 스레드/메시지 상세 화면
- 통계 대시보드, 그래프, 고급 필터

### 관리자 로그인 페이지

> **구현 기능:** `F010` | **메뉴 위치:** 인증 섹션 (비로그인 시 표시)

| 항목 | 내용 |
|------|------|
| **역할** | 시스템 관리자 전용 인증 진입점 |
| **진입 경로** | 초기 접속 시 자동 진입, 미인증 상태에서 보호된 페이지 접근 시 자동 리디렉션 |

| **주요 기능** | • 이메일 형식 검증 (Zod)<br>• 비밀번호 입력 필드<br>• Supabase Auth 로그인 호출<br>• 인증 실패 시 에러 메시지 표시<br>• **로그인** 버튼 |
| **다음 이동** | 성공 → 테넌트 관리 페이지, 실패 → 에러 메시지 표시 후 페이지 유지 |

## 🗄️ 데이터 모델

### MVP 직접 사용 테이블

#### tenant (멀티테넌트 관리)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 테넌트 고유 식별자 | UUID |
| name | 테넌트(입점사·가맹점) 이름 | Text |
| type | 테넌트 유형 (입점사/가맹점) | Text |
| status | 활성/비활성 상태 | Text |
| created_at | 생성 시각 | Timestamp |

#### user (사용자)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| user_id | 사용자 고유 식별자 | UUID |
| auth_user_id | Supabase Auth UID (로그인 연동 키) | UUID |
| tenant_id | 소속 테넌트 (nullable: 플랫폼 관리자) | → tenant.id |
| email | 사용자 이메일 (로그인 ID) | Text |
| name | 사용자 이름 | Text |
| phone | 전화번호 (nullable) | Text |
| role | 사용자 역할 (ADMIN / SELLER / CUSTOMER / RIDER) | Text |
| active | 활성 여부 | Boolean |

> **주의**: `department`(소속) 컬럼은 users 테이블에 존재하지 않습니다. UI 표시 시 role을 한국어로 변환하여 표시합니다.

#### common_code (공통코드 마스터)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 공통코드 마스터 고유 식별자 | UUID |
| code | 마스터 코드 값 | Text |
| name | 마스터 코드 이름 | Text |
| description | 마스터 코드 설명 | Text |

#### common_code_value (공통코드 상세)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 공통코드 상세 고유 식별자 | UUID |
| common_code_id | 소속 마스터 코드 | → common_code.id |
| value | 코드값 | Text |
| label | 코드 라벨 | Text |
| sort_order | 정렬 순서 | Integer |

#### fp_ad_placement (광고 위치·이미지)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 광고 고유 식별자 | UUID |
| tenant_id | 광고 소유 테넌트 | → tenant.id |
| position | 광고 노출 위치 | Text |
| image_url | 광고 이미지 URL (Supabase Storage) | Text |
| is_active | 활성화 여부 | Boolean |

## 🛠️ 기술 스택 (공식 문서 최신 버전 — 2026-04-13 기준)

> 각 항목은 공식 문서 설치 가이드와 프로젝트 `package.json`을 교차 검증한 값입니다.

### 🎨 프론트엔드 프레임워크

- **Next.js 16** (App Router, Turbopack 기본 번들러)
  - 공식: <https://nextjs.org/docs/app/getting-started/installation> (v16.2.3)
  - 최소 요구사항: Node.js 20.9+
- **React 19.2** — UI 라이브러리 (최신 동시성 기능)
  - 공식: <https://react.dev/versions>
- **TypeScript 5.x** (최소 5.1.0) — 타입 안전성 보장

### 🎨 스타일링 & UI

- **TailwindCSS v4.2** — 설정 파일 없이 `@tailwindcss/postcss` + `postcss.config.mjs` + `@import "tailwindcss"` 구조
  - 공식: <https://tailwindcss.com/docs/installation/framework-guides/nextjs>
- **shadcn/ui** (new-york 스타일) — Radix UI 기반 컴포넌트
  - 공식: <https://ui.shadcn.com/docs/installation/next>
  - CLI: `npx shadcn@latest init` / `npx shadcn@latest add [component]`
- **Lucide React** — 아이콘 라이브러리

### 📝 폼 & 검증

- **React Hook Form 7.x** — 폼 상태 관리
- **Zod 4.x** — 스키마 검증 (`@hookform/resolvers`로 연결)

### 🗄️ 백엔드 & 데이터베이스 (Supabase)

- **@supabase/ssr + @supabase/supabase-js** — Next.js App Router 공식 권장 조합
  - 공식: <https://supabase.com/docs/guides/auth/server-side/nextjs>
  - `createServerClient` (서버 컴포넌트/Route Handler) / `createBrowserClient` (클라이언트 컴포넌트)
- **Supabase Auth** — 이메일/비밀번호 기반 관리자 인증
- **PostgreSQL** — Supabase 관리형 관계형 데이터베이스
